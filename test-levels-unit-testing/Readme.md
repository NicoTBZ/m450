# Ergebnisse: Nico (OPTEN AG) & Ardit (CMI AG)

## Aufgabe 1

### Nico – OPTEN AG
- Entwickeln Webauftritte; getestet wird vor allem **manuell im Frontend (Blackbox)**.  
- **Keine QA-Teams**, Tests machen Entwickler oder Kunden.  
- Tests finden meist **am Ende der Entwicklung** statt.  
- **Life Cycle:** Anforderungen → Entwicklung → Manuelle Tests → Anpassungen → Deployment.

### Ardit – CMI AG
- Entwickeln umfangreiche Software fuer Gemeinden und Schulen.  
- Nutzen **Unit-, Integrations- und Cucumber-Tests** sowie **BBD**.  
- Tests laufen **kontinuierlich** im CI/CD.  
- **Dediziertes QA-Team**.  
- **Life Cycle:** BBD-Szenarien → Entwicklung + Tests → Automatisierte Tests → QA-Freigabe → Release.

---

## Aufgabe 2 – Einordnung der Begriffe

### Testing Approach
- Grundsaetzliche Strategie, wie getestet wird  
  (z. B. Blackbox, BBD, Exploratory).

### Testing Levels
- Ebenen des Testens:  
  **Unit → Integration → System → Acceptance**.

### Testing Types, Techniques und Tactics
- **Types:** funktional, nichtfunktional  
- **Techniques:** z. B. Boundary Analysis, Equivalence Partitioning  
- **Tactics:** konkrete Mittel wie Mocking, Stubbing, Cucumber

**Abhaengigkeit:**  
Approach bestimmt den Stil → Levels definieren die Ebene → Types/Techniques/Tactics bestimmen das konkrete Vorgehen.
